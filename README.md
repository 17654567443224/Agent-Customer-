# Agent-Customer 电商智能客服系统

**Agent-Customer** 是一款基于 PyQt6 构建的电商 AI 客服桌面应用。它通过集成大语言模型 (LLM) 与自研 Agent 框架，为电商场景提供智能、自动的客服回复能力，并支持多平台渠道接入。

## ✨ 核心功能

- **多渠道消息集成**：目前已支持通过 WebSocket 实时接收拼多多平台消息，为后续扩展其他电商平台预留了架构。
- **AI 智能客服代理**：
  - 基于**自研 Agent 框架**（不依赖 Agno）构建，支持多轮工具调用与完整的会话上下文管理。
  - AI 代理可主动向用户推荐商品，通过工具获取商品列表并发送商品卡片。
- **双知识库体系**：
  - **产品知识库**：存储并检索商品规格、用法、价格等详细信息。
  - **客服知识库**：专门用于回答售后、物流、退款等政策类问题。
  - 支持从拼多多 API 拉取商品列表，并调用多模态 LLM 自动提取知识入库。
- **智能会话管理**：
  - **关键词自动转人工**：可根据配置的关键词，自动将复杂或敏感会话转接给人工客服。
  - **异步消息队列**：采用异步消息队列与处理器链，确保高并发场景下的消息处理效率。
  - **稳定连接**：WebSocket 连接具备自动重连与心跳检测机制，保证服务稳定性。

### 🛠️ AI 代理工具集

Agent 通过以下工具与外部交互：

| 工具名称 | 功能描述 |
| :--- | :--- |
| `get_shop_products` | 按价格、销量、库存、标签等条件查询店铺商品列表。 |
| `send_goods_link` | 向当前对话用户发送指定的商品卡片链接。 |
| `get_product_knowledge` | 查询特定商品的详细知识（如成分、规格、用法）。 |
| `search_customer_service_knowledge` | 在客服知识库中搜索售后、物流、退款等政策答案。 |
| `transfer_conversation` | 将会话转接给人工客服处理。 |
